# AWEBO Launchpad – UX Review + Frontend Scope (Sprint 1)

**Budget:** $400 USD  
**Scope Type:** UX Audit + Core UI Implementation (Public User Flow Only)  
**Purpose:** Define exactly what is delivered in this sprint and what is deferred.

---

# 1. Objective of This Sprint

This sprint focuses on:

- Improving conversion clarity
- Strengthening trust in token launches
- Clarifying the phygital (token + merch) value proposition
- Delivering a clean, responsive frontend foundation
- Building the core public launch experience

This is **not a full platform build**.

---

# 2. Deliverables (In Scope)

## 2.1 UX Audit Deliverable

A concise UX Audit Document including:

- Conversion flow analysis (Landing → Launch → Participation CTA)
- Friction identification
- Clarity improvements for token + merch relationship
- Wallet UX simplification recommendations
- Trust-building improvement suggestions
- CTA hierarchy & positioning recommendations
- 5–10 prioritized action items

Output format:

- Markdown or PDF document
- Structured and actionable
- Not a full redesign specification

---

## 2.2 Frontend Implementation (Public Side Only)

### A. Landing Page (Simplified)

Includes:

- Hero section
- Clear value proposition
- Featured launch section (static/mock data)
- Primary CTA buttons
- Responsive layout

Does NOT include:

- CMS
- Filters
- Animations beyond basic CSS
- SEO optimization

---

### B. Launch Detail Page (Primary Focus)

Core page of this sprint.

Includes:

- Project hero section
- Token info panel (mock data)
- Static countdown UI
- Static progress bar
- Participation CTA states (UI only)
- Phygital explanation section
- Merch preview gallery (static images)
- Digital vs Phygital tier comparison block

Does NOT include:

- Live token logic
- Real countdown sync
- Blockchain interaction
- Allocation calculations

---

### C. Wallet & Network UI (Stub Only)

Includes:

- Wallet connect modal (UI only)
- Connected / not connected states
- Wrong network state
- Sold out state

Does NOT include:

- Real wallet integration
- Transaction signing
- Multichain switching logic

---

### D. Basic Design System Foundation

Includes:

- Button styles
- Card components
- Badge components
- Typography scale
- Color system
- Layout structure

Does NOT include:

- Full component library
- Storybook
- Extensive theming system

---

### E. Responsiveness

Includes:

- Mobile optimization for Landing
- Mobile optimization for Launch page
- Basic cross-browser check

---

# 3. Explicitly Out of Scope (Deferred to Future Sprint)

The following are NOT included in this engagement:

- Smart contracts
- Backend or API development
- Database integration
- Real wallet integration
- Tokenomics logic
- Shipping & logistics logic
- User profile system
- Launch creator/admin dashboard
- Dynamic filters
- CMS/blog
- DevOps or production deployment
- Advanced animations
- Full QA testing suite
- Ongoing maintenance

These items require a separate sprint and budget.

---

# 4. Technical Assumptions

- Framework: Next.js
- Styling: Tailwind
- Data: Mock/static only
- Web3: UI stubs only
- No backend integration

---

# 5. Definition of Done (Sprint 1)

This sprint is considered complete when:

- Landing page is responsive and polished
- Launch detail page is visually complete
- Wallet UI states are implemented (static)
- UX audit document is delivered
- Codebase is clean and organized
- No runtime errors in local environment

---

# 6. Future Sprint Candidates

Potential Sprint 2+ items:

- Real wallet integration
- Multichain logic
- Backend API
- Launch creator dashboard
- User profile
- Real allocation logic
- Live countdown
- Shipping system integration
- CMS for launches
- Analytics integration

---

# Summary

This sprint delivers a **high-quality public-facing prototype and UX clarity pass**, not a production-ready Web3 platform.

The primary focus is the **Launch Detail Page experience** and clear positioning of AWEBO’s phygital differentiation.
